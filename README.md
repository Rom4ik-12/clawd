**Clawd** — это продвинутый селф-агент на базе Telethon и языковых моделей (LLM), работающий непосредственно на вашем Telegram-аккаунте (юзербот). Он совмещает функции автономного ИИ-ассистента и интерактивного помощника, способного выполнять задачи на вашем сервере, искать информацию в интернете и автоматизировать работу с Telegram.

Управление ботом осуществляется через **Панель управления (отдельный Telegram-бот)**, что исключает необходимость прямого доступа к серверу для базовых настроек.

---

## 🌟 Основные возможности

### 🧠 Агентный цикл (Agent Loop)
* **Автономность**: Способность планировать действия и вызывать цепочки инструментов для достижения поставленной цели.
* **Мультимодельность**: Поддержка текстовых моделей и моделей зрения (Vision) через OpenRouter, Codex.sale, Runic или любой OpenAI-совместимый эндпоинт (Ollama, LMStudio, Groq).

### 🛠 Встроенные инструменты (Tools)
* **Работа с системой**: Выполнение `bash`/`python` команд прямо на сервере, чтение/запись файлов, просмотр директорий, скачивание файлов по URL, мониторинг системных ресурсов (CPU, RAM, Uptime).
* **Поиск в сети**: Интеграция с поисковой системой **SearXNG** для получения актуальных данных в реальном времени.
* **Автоматизация Telegram**:
  * Чтение ленты каналов, чатов и профилей.
  * Отправка сообщений, управление подписками на каналы (join/leave).
  * Взаимодействие с другими ботами (нажатие инлайн-кнопок).
  * Отправка реакций и стикеров.
* **Планировщик задач**: Отложенный или периодический запуск действий (например, ежедневный дайджест новостей или выполнение фоновых задач).

### 🎛 Панель управления (Telegram-бот)
Управляйте своим юзерботом удаленно:
1. **Профиль**: Просмотр и редактирование имени в Telegram, описания («О себе») и аватара.
2. **Конфигурация**: Переключение активной модели LLM в один клик, смена системного имени бота, настройка триггерных слов в группах, выбор языка общения.
3. **Безопасность**: Интерактивная консоль для выполнения shell-команд с просмотром истории последних 10 команд.
4. **Управление памятью**: Очистка истории диалогов и контекста для конкретных пользователей или чатов.

## 📂 Структура проекта

```text
├── automation/          # Автоматические задачи и планировщик
├── brain/               # Ядро агента (агентный цикл, формирование контекста)
├── database/            # Локальные базы данных SQLite и векторные БД (создаются при старте)
├── host/                # Инструменты взаимодействия с сервером (выполнение shell, файлы)
├── internet/            # Модули интеграции с веб-сервисами (SearXNG, парсинг Telegram)
├── llm/                 # Коннекторы к провайдерам LLM и системные промпты
├── logs/                # Локальные логи (создаются автоматически)
├── memory/              # Менеджеры работы с базой данных и памятью бота
├── tg/                  # Логика работы юзербота (Telethon) и панели управления (бот)
├── .env.example         # Пример файла конфигурации
├── .gitignore           # Игнорируемые Git файлы
├── main.py              # Точка входа для запуска
└── requirements.txt     # Зависимости Python
```

## ⚙️ Быстрый запуск

### 1. Подготовка окружения
Клонируйте репозиторий и перейдите в папку проекта:
```bash
git clone <url-вашего-репозитория>
cd Clawd
```

Создайте и активируйте виртуальное окружение Python:
```bash
python3 -m venv venv
source venv/bin/activate
```

Установите необходимые зависимости:
```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
Скопируйте `.env.example` в `.env`:
```bash
cp .env.example .env
```

Откройте `.env` и заполните обязательные параметры:
* `TG_API_ID` и `TG_API_HASH` — получите их на [my.telegram.org](https://my.telegram.org/).
* `PANEL_BOT_TOKEN` — токен для бота панели управления (создайте через [@BotFather](https://t.me/BotFather)).
* `OWNER_ID` — ваш числовой Telegram ID (для авторизации в панели управления и юзерботе).
* `OPENROUTER_API_KEY` или `CODEXSALE_API_KEY` — ключ API к провайдеру LLM.

### 3. Первый запуск и авторизация
Запустите основное приложение:
```bash
python3 main.py
```
При первом запуске консоль юзербота попросит вас ввести номер телефона и код подтверждения от Telegram для авторизации сессии юзербота. После успешного входа сессия сохранится в файле `clawd.session`.

## 🔒 Безопасность при публикации
Файл `.gitignore` настроен на игнорирование всех конфиденциальных файлов:
* Файлы сессий Telegram (`*.session`, `*.session-journal`).
* Базы данных и кэш (`database/*.db`, `database/vector_db/`).
* Локальные логи (`logs/`, `*.log`).
* Конфигурационный файл с ключами API (`.env`).

**НИКОГДА** не выкладывайте файл `.env` или `.session` в публичный доступ!
